# Security Policy

## Supported Versions

| Version | Supported |
| --- | --- |
| 0.1.x | Yes |

## Reporting a Vulnerability

Use GitHub private vulnerability reporting from the repository's **Security**
tab and select **Report a vulnerability**. Do not open a public issue for a
security report.

Never include Telegram API hashes, session strings, administrator passwords,
cookies, proxy credentials, private download paths, or release-safety denylist
values in public issues, discussions, or logs.

Rotate Telegram API credentials and session files if they were committed,
shared, or exposed in logs.
