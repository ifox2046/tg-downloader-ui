<completed_work>

- Added a separate `pause_requested` database field and explicit `paused` job status.
- Active Pause now asks the existing worker-owned `Popen` process to receive `SIGSTOP`.
- Continue sends `SIGCONT` to the same process and restores its prior worker status.
- Cancel and service shutdown send `SIGCONT` before `SIGINT`, allowing stopped children to terminate cleanly.
- Signals are sent through the live `Popen` object rather than a database PID, preventing PID-reuse mistakes.
- Added a process-level regression test proving the PID remains the same, partial bytes do not change while paused, and bytes continue increasing after resume.
- Updated the web UI to distinguish `paused` from `canceled`.
- Documented the distinction between live pause and restart recovery.
- Built the final IPK at `.tmp-openwrt-pause-resume-final/tg-downloader-ui_0.1.0_all.ipk`.

</completed_work>

<decisions_made>

- Live Pause/Continue uses `SIGSTOP/SIGCONT` on the worker-owned process object.
- `tdl --continue` remains only for skipping completed items after relaunch; it is not described as single-file byte resume.
- Queued jobs use the explicit `paused` state and resume back to `queued`.
- Cancel and shutdown always continue a potentially stopped child before interrupting it.

</decisions_made>
